Реализовать gRPC сервер, который будет запускаться в локальной сети и принимать от
мобильного клиента телеметрию с данными, описанными в структуре SessionData
(файл session.proto в репозитории https://gitlab.com/actant.app/common/grpc-interfaces)
и выводить данные в консоль в текстовом виде (формат произвольный).

Также сервер должен отправлять часть этих данных в бинарном виде на локальный
 сокет (порт задаётся в аргументах запуска). Для примера взять данные,
 приходящие с клиента в proxy.proto:TelemetryService.InStream,
 конкретнее облака точек (SessionStreamMessage.data_frame.ar_session_frame.point_cloud.points)
 и инерциальные данные (SessionStreamMessage.motion_frame)

Сервер должен запускаться на чистой Ubuntu 18.04.4 LTS Desktop с
установкой всех необходимых зависимостей в docker-контейнере

Результат должен быть передан в виде архива с кодом либо в виде
ссылки на собственный репозиторий. Оригинальные репозитории не должны изменяться.

Ссылка на репозиторий Android-клиента: https://gitlab.com/actant-android/ar-android/-/tree/feature/remove_opencv
